🏷️ 二、版本号策略（Semantic Versioning）
采用 语义化版本（SemVer）：MAJOR.MINOR.PATCH

版本段	含义	触发条件
MAJOR	不兼容变更	架构重构、数据库表结构破坏性变更
MINOR	新功能	新增模块（如 factor_fetcher, index_fetcher）
PATCH	修复/优化	Bug 修复、性能提升、文档完善
当前版本路线图
版本	状态	内容
v0.1.0	✅ 今日发布	架构奠基：日线下载 + 股票池 + 分库设计
v0.2.0	🚧 下一目标	复权因子下载 + 指数数据接入
v0.3.0	⏳ 计划中	指标计算引擎 + 合并输出模块
v1.0.0	🎯 里程碑	首个稳定版：支持完整日线+复权+指标回测输入
💡 为什么从 v0.1.0 开始？

v0.x 表示“初始开发阶段”，API/结构可能调整
达到 v1.0.0 时表示基础数据流稳定，可供策略团队依赖

🔖 四、打标签（Tag）正式版本
每次发布 MINOR/PATCH 版本时打 Git Tag：

powershell
编辑

✅ 今日 Git 操作总结
步骤	命令
1. 初始化仓库	git init
2. 创建 .gitignore	（内容见上）
3. 添加所有代码	git add .
4. 首次提交	git commit -m "feat: init ..."
5. 打版本标签	git tag -a v0.1.0 -m "..."
🎯 后续 Git 实践建议
分支策略：
main：始终可运行的稳定版
dev：日常开发（合并 feature 后测通过再入 main）
提交规范：
使用 Conventional Commits：
feat: 新功能
fix: 修复
docs: 文档
